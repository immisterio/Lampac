# Middlewares (пример)

Учебный модуль в пространстве имён **`Lamson`**: демонстрация цепочки **`EventListener.Middleware`** в Lampac — подписываются **два** обработчика (оба имеют сигнатуру **`Task<bool> …(bool first, EventMiddleware e)`**), вызываются в порядке подписки. В **`Dispose`** обе подписки снимаются.

## Первый обработчик `Invoke`

Параметр **`first`** — признак первого прохода middleware.

- Если **`first`** **или** запрос локальный (**`IsLocalRequest`**) **или** анонимный (**`IsAnonymousRequest`**) → сразу **`true`** (пропуск без заголовка).
- Иначе, если путь совпадает с **`^/(kinogram|porngram|lamson)`**, в ответ добавляется заголовок **`X-Lamson: Middleware was here`**.
- Возвращается **`true`** — пайплайн продолжается.

## Второй обработчик `InvokeAsync`

- Если **`first == false`** → сразу **`true`** (на последующих проходах ничего не делает).
- Если путь начинается с **`/lamson`** (без учёта регистра):
  - нет непустого query-параметра **`token`** → ответ JSON **`[{"error":"token == null"}]`**, возврат **`false`** (остановка пайплайна);
  - иначе проверка «VIP» (заглушка **`isvip = true`**; в коде закомментирован пример HTTP к своему API) — при успехе **`true`**.
- Любой другой исход на **`first == true`** (в т.ч. путь **не** под **`/lamson`**, либо VIP не прошёл) → ответ JSON **`[{"error":"not vip"}]`**, **`false`**.

## manifest.json

- **`dynamic`: true** — модуль компилируется Roslyn при загрузке.
- В **`tree`** только **`ModInit.cs`** — отдельных контроллеров нет.

## Назначение

Пример для своих проверок доступа и заголовков; перед продакшеном продумайте порядок middleware и не блокируйте служебные пути без необходимости.
