# DLNA

**DLNA** внутри Lampac: каталог торрентов/раздач, стриминг на устройства по совместимым шаблонам, управление торрент-клиентом (старт/пауза/приоритет файлов), подгрузка трекеров в magnet и JS-плагин для клиента.

## Назначение

- Инициализирует **`DLNAController`** при загрузке модуля.
- Запускает **`TrackersCron`** для периодического обновления списков трекеров (если включено в конфиге).
- Хранит пользовательские данные относительно **`path`** (по умолчанию `data/dlna`).

## Конфигурация

Секция в `init.conf`: **`DLNA`**.

Из коробки в `ModInit` задаются:

- **`path`** — корень данных;
- **`uploadSpeed`**, **`autoupdatetrackers`**, **`intervalUpdateTrackers`**;
- **`addTrackersToMagnet`** — добавлять ли announce URL из списка трекеров к magnet;
- **`mediaPattern`** — расширения файлов для медиа;
- **`limit_map`** — WAF для **`^/dlna/`**.

## HTTP / JS (обзор)

| Шаблон маршрута | Назначение |
|-----------------|------------|
| `dlna.js`, `dlna/js/{token}` | Плагин интерфейса. |
| `/dlna` | Основной UI/логика списка (см. `Controller.cs`). |
| `/dlna/stream` | Отдача потока. |
| `/dlna/delete` | Удаление элементов. |
| `/dlna/tracker/*` | Операции с торрентами: менеджеры, загрузка, старт/стоп/пауза, приоритет файла. |

Полный перечень и параметры — в **`Controller.cs`** (файл большой, разделы помечены `[Route]`).

## Зависимости

- Рабочий торрент-слой и интеграция с тем же backend, что ожидает `DLNAController` (детали в Shared/хосте).

## Выгрузка

При `Dispose` останавливается **`TrackersCron`**.
