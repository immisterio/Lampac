# SyncEvents

Лёгкий модуль, который поднимает **полную** подсистему WebSocket **`NwsEvents`** (проект **SyncEvents**, namespace **`SyncEvents`**).

Параметр **`onlyreg: false`** включает **полный** набор обработчиков сообщений, а не только регистрацию клиентов.

## Зачем отдельный модуль

Модуль **Sync** вызывает **`NwsEvents.Start(onlyreg: true)`** — режим «в основном регистрация». Без **SyncEvents** полная шина событий может быть неактивна. Типичная связка: **Sync** + **SyncEvents** + при необходимости **Storage**.

## Конфигурация

Секции в `init.conf` **нет** — только **`manifest.json`** (**`enable`**).

## Жизненный цикл

| Этап | Действие |
|------|----------|
| **`Loaded`** | **`NwsEvents.Start(onlyreg: false)`** |
| **`Dispose`** | **`NwsEvents.Stop()`** |

## Зависимости

- Класс **`NwsEvents`** объявлен в проекте **SyncEvents** и **используется тем же именем** из модуля **Sync** (`using SyncEvents`) — это **одна** реализация на процесс; порядок старта/остановки модулей задаёт хост.
- Имеет смысл включать, когда нужны не только WebSocket-«регистрации», но и полная обработка сообщений синхронизации.

## Интерфейс

**`IModuleLoaded`** — без **`IModuleConfigure`** и без собственных контроллеров.
