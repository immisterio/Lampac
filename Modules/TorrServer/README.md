# TorrServer

Встраивает **[TorrServer](https://github.com/YouROK/TorrServer)** в Lampac: скачивает бинарник под ОС/архитектуру, держит процесс, проксирует веб-UI и API под префиксом **`/ts`**, выдаёт **`ts.js`** для клиента.

## Назначение

- Данные и бинарник лежат в **`data/ts/`** (от корня рабочей директории процесса): `TorrServer-windows-amd64.exe` или `TorrServer-linux-*`.
- При первом запуске копируется **`settings.json`** из каталога модуля; пароль процесса совпадает с **`tspass`** (генерируется при старте), записывается в **`accs.db`** рядом с данными.
- Фоновый поток **скачивает/обновляет** релиз с GitHub (`releases` в конфиге — тег или **`latest`**), выставляет **`chmod +x`** на Linux.
- HTTP-клиент контроллера ходит на **`http://{listen.localhost}:{tsport}`** с Basic **`ts:{tspass}`**.

## HTTP (обзор)

| Маршрут | Описание |
|---------|----------|
| `GET ts.js`, `ts/js/{token}` | Плагин TorrServer для Lampa; опционально подставляет `torrserver_login` из token. |
| `GET /ts`, `/ts/static/js/{suffix}` | HTML/JS веб-интерфейса TorrServer с подменой путей на префикс `/ts/`. |
| `GET/POST /ts/{*suffix}` | Прокси на API TorrServer; для потоков `/ts/stream` и `/ts/play` при включённом **accsdb** — особая логика авторизации (см. `Controller.cs`). |

В белый список базовых путей модулей добавлено **`/ts/`**.

## Конфигурация

Секция в `init.conf`: **`TorrServer`** (`ModuleConf`).

| Поле | Смысл |
|------|--------|
| `tsport` | Порт локального TorrServer (по умолчанию **9085**). |
| `releases` | Версия GitHub или **`latest`**. |
| `defaultPasswd` | Служебный пароль по умолчанию в конфиге модуля (**не** равен динамическому `tspass` без проверки кода). |
| `checkfile` | Участвует в логике установки/проверки бинарника. |
| `rdb` | Флаг конфигурации расширенного режима (см. использование в коде). |
| `group` | Группировка в UI/настройках. |
| `limit_map` | WAF для **`^/ts/`**. |

## Зависимости

- Сеть для загрузки релиза с GitHub.
- На Linux — доступ к **`uname`** для выбора архитектуры.

## Остановка

При выгрузке модуля процесс TorrServer принудительно завершается (`Dispose`).
